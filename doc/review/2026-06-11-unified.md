2. **Architecture formula (operator, evening): "botfam is now IRC + bots +
   local sandbox-only shims."** Corollary invariant: protocol surfaces live
   on IRC; host-local mechanisms (signals, pidfiles, flocks) may exist only

## 6. Lessons Learned

### L1: Unverified state presented with authority is one failure class, not four
Evidence: stale `~/bin/botfam` reporting the wrong SHA (16:31); meta.ai
prescribing for a maildir/no-daemon botfam that no longer exists; Gemini
synthesizing a fictional postmortem from stale chat memory; the judge's
stale grep accusing qwen of inventing a real PID.
Protocol implication: provenance checks at every boundary — binaries
announce embedded SHAs, reviewers get current ground truth, reviews get
transcript verification, accusations get the same evidence bar as claims.
Code/tooling implication: `!version` (done); ground-truth section in the
review prompt (done); CI check that BuildInfo is non-empty (open, AI-R9).

### L2: Governance that can ratify after the fact will
Evidence: F1 + F2 — and F2 reproduced *during this retro's own merge*
(19-second approval), proving the pattern is structural, not situational.
Protocol implication: a `risk=prod` proposal tier whose `!executed` is
invalid unless an APPROVED tally precedes execution; operator verbal
instructions stay legal but get labeled (`override=operator`) so the
transcript distinguishes operator-gated from quorum-gated.
Code/tooling implication: scribe validates event ordering for prod-tier
proposals; approval messages on prod tier require an `evidence=` field.

### L3: Structure and insight trade off in review prompts — buy both
Evidence: the Gemini A/B. Structured prompt: extractable, costed, typed
(9/10); free-form: found the liveness-perception failure class the
structured one never looked for (8/10). v2 adds a mandated Blind-spots
section to buy insight back inside the structure.
Protocol implication: prompts are versioned fam property; changes re-run
the A/B before being trusted (provenance note in the prompt).

### L4: Cheap cold reviewers are an ensemble, not a downgrade
Evidence: gemma4 (scribe silent-stall healthcheck) and qwen3.5 (retention
parity) each found something all four frontier reviewers missed, while
being individually less reliable. The panel's value came from union +
per-review verification, not from any single reviewer's quality.
Protocol implication: review rounds should prefer N cheap diverse
reviewers + mandatory transcript verification over one blessed reviewer.

### L5: The watcher is infrastructure, not a loop the agent remembers to run
Evidence: F7 — agy went silent twice because `irc-wait` fires once and
re-arming is the agent's job; the operator had to hand-ping.
Protocol implication: liveness must be observable on the substrate (nick
presence + watcher-answered `!ping`), not inferred from silence.
Code/tooling implication: resident watcher (AI-R4) that holds the IRC
connection, auto-re-arms, and answers `!ping` from the process — closing
F7 and gemma4's race window in one design.

## 5. Lessons Learned

### Lesson 1: Running code is not reviewed code — deploys are protocol state

Evidence from this session:

- Scribe answered tallies from a stale binary twice (10:32, 13:51), both times
  immediately after the code it embodied had been changed.

Protocol implication:

- The executor's landing checklist must include restarting affected bots and
  verifying the running version, or the consensus machinery silently runs old
  rules.

Code/tooling implication:

- AI-5 (`!version` answering with build sha) is the verification primitive;
  promote from P2 to P1.

### Lesson 2: Port liveness is not server identity

Evidence from this session:

- 11:03: ngircd (wildcard bind) and ergo (specific loopback binds) coexisted on
  port 6667; nc reported the port "up" while two disjoint #botfam channels
  existed. agy would have rejoined into an empty room.

Protocol implication:

- A server migration step must verify there is exactly one listener (lsof by
  pid), not just that the port answers.

Code/tooling implication:

- Add a single-listener check to any future server-swap runbook/script.

### Lesson 3: Literal command scripts beat prose intents for waking agents into in-flight work

Evidence from this session:

- The first fresh agy "came up with a new implementation plan" (operator
  report, ~13:42); the second, given a numbered command script with an explicit
  "do NOT re-plan" header, executed it verbatim including the announce line
  (13:46:03), and the fixes landed in 3 commits in 2 minutes.

Protocol implication:

- Wake-up prompts for agents joining in-flight work should be executable
  (commands + acceptance criteria), not descriptive.

### Lesson 4: Substrate durability beats convention durability

Evidence from this session:

- The replay-on-join *convention* required every client to implement reading a
  shared ledger; ergo's CHATHISTORY made recovery automatic, and it absorbed
  both the machine-sleep outage (13:39, rlupi's "Hi") and routine reconnects
  with zero per-agent code beyond CAP negotiation.

Protocol implication:

- Prefer substrate features over conventions for safety properties; keep the
  scribe ledger as the machine-readable ccrep record, not the recovery path.

### Lesson 5: A human in the loop is also a lossy client

Evidence from this session:

- The operator's 11:08 instructions never reached the channel (ircII status
  window); claude's morning channel messages reached him only after he joined
  the right server. Both directions of human-agent delivery failed silently at
  least once today.

Protocol implication:

- Decision-relevant operator instructions received out-of-band (GUI) must be
  echoed into the channel by the receiving agent — done manually today (10:26
  yolo relay, 11:08 GUI relay); should be a standing rule.

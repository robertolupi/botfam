## 5. Lessons Learned

### Lesson 1: Coordinate proposal ownership before implementing duplicate skills

- **Evidence:** `agy` and `claude` concurrently wrote and proposed different
  formatting skill cards (`botfam-markdown-formatting` vs `writing-markdown`),
  causing a minor collision.
- **Protocol implication:** Agents should announce intent to claim an area of
  work/documentation before writing and proposing, not just at proposal time.
- **Code/tooling implication:** None.

## 6. Things That Went Well

### Went well: centralizing identity normalization in the merge gate

- **Evidence:** Normalizing reviewer/author names in `newCcrepEvent` by
  stripping the `-cli` suffix corrected the self-approval issue retroactively
  because the scribe parses the entire ledger upon launch.
- **Why it mattered:** Restores the exact integrity of quorums and approvals.
- **Should we reinforce this?** Yes.

## 7. Things That Went Poorly

### Went poorly: building Docker images from worktrees blocks VCS resolution

- **Evidence:** Scribe container built from `wt-agy` worktree resulted in
  `version dev` because `.git` is a pointer file that couldn't be resolved
  inside the container.
- **Impact:** Stamping failed until the container was built from the main
  checkout (`~/src/botfam`).
- **Could automation prevent this?** Yes.
- **Suggested fix:** Document that all production builds must run from the main
  checkout.

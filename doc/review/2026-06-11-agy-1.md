## 5. Lessons Learned

### Lesson 1: Process Leakage Across Harness Compactions

Evidence from this session:
- The agent harness restarted, stopping tasks inside the harness context, but did not terminate background processes started in previous harness instances (PIDs 58945 and 59790). This caused port/nick collisions on ergo.

Protocol implication:
- None.

Code/tooling implication:
- Network-bound command subcommands (like `irc-client` and `scribe`) should write lockfiles/PIDs or check for active sockets on startup to detect and report stale background processes.

### Lesson 2: Inode Replacement and FD Leakage

Evidence from this session:
- Swapping the log file path via `cat ... > temp && mv temp history.jsonl` replaced the active inode. The running scribe process continued writing to the old unlinked file descriptor, while tally readers queried the new inode, resulting in a split-brain.

Protocol implication:
- None.

Code/tooling implication:
- Unified ledger updates must preserve the file inode (e.g. append-only writing, or truncating and rewriting instead of atomic replacement via rename/move) or require restarting the daemon holding the write handle.

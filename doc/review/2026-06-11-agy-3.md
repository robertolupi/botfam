## 5. Lessons Learned

### Lesson 1: Avoid Executing External Tools for Runtime Fallbacks

- **Evidence from this session**: The previous `GetVersion()` implementation
  executed `git rev-parse HEAD`. In directories without a git history or where
  the `.git` directory structure was complex, it could either fail or misreport
  a stale version.
- **Protocol implication**: Stamping and versioning must rely on
  self-contained, reproducible builds.
- **Code/tooling implication**: Using `debug.ReadBuildInfo()` settings
  (`vcs.revision` and `vcs.modified`) provides a clean, zero-dependency version
  indicator for Go binaries.

### Lesson 2: Ensure Goroutines Reading from Blocking Resources are Cancelable

- **Evidence from this session**: When the IRC connection was severed, the main
  thread exited the read loop and waited for `done`. However, the scanner
  goroutine reading from the FIFO file remained blocked, causing the client
  process to hang forever.
- **Protocol implication**: Network clients must handle abrupt disconnections
  and terminate cleanly.
- **Code/tooling implication**: The FIFO reader should use a non-blocking
  check, check for a cancellation context, or the main thread should close the
  FIFO file descriptor when breaking out of the socket read loop to unblock the
  scanner.

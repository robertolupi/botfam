## 5. Lessons Learned

### Lesson 1: Avoid Executing External Tools for Runtime Fallbacks
- **Evidence from this session**: The previous `GetVersion()` implementation executed `git rev-parse HEAD`. In directories without a git history or where the `.git` directory structure was complex, it could either fail or misreport a stale version.
- **Protocol implication**: Stamping and versioning must rely on self-contained, reproducible builds.
- **Code/tooling implication**: Using `debug.ReadBuildInfo()` settings (`vcs.revision` and `vcs.modified`) provides a clean, zero-dependency version indicator for Go binaries.

- **Impact**: Execution paused until manual user intervention.
- **Likely system cause**: macOS's strict protection over `~/Library` and general application sandboxing rules.
- **Could automation prevent this?** No, but documentation/warnings can prepare operators.
